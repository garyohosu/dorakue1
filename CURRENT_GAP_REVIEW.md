## 2. 注意が必要な項目 (Needs Attention)
*   **ブラウザの自動再生ポリシー**:
    *   Web Audio API 音源を使用しているため、ブラウザ起動直後に勝手にBGMを再生しようとすると再生制限（Autoplay Policy）により無音になってしまいます。
    *   [main.js](file:///C:/project/dorakue1/src/main.js) に初回クリック/キー入力を監視して [initAudio()](file:///C:/project/dorakue1/src/audio/sfx.js#L22) を呼ぶ処理がすでに記述されているため、Phaserの各シーン（特に TitleScene）ではこれらが有効化された後に音響メソッドを呼び出す設計を徹底してください。
*   **ドラクエなどの商標・デザイン模倣排除**:
    *   ゲーム内の名称（リム、アレン、オルヴェスなど）や「攻める」「術法」などの非伝統的コマンド名は仕様通りに徹底し、戦闘メッセージなどもDQ風に酷似しすぎないように実装します。

## 3. 次回最優先で実装すべきこと (Next Phase Priority)
次回作業再開時は、**「TitleScene」から「FieldScene」への遷移**、および**「フィールド上でのアレンの1マス移動と通行判定」**の2つを最優先のMVP（最小動作確認版）として開発します。
*   画面上にテキストだけでタイトルメニューを表示。
*   キー操作でメニューを選択し、移動時にカーソル音（sfx.js）、決定時に決定音（sfx.js）が鳴り、FieldSceneへ遷移する。
*   FieldSceneでは、四角形の仮主人公アレンをキーボードで移動させ、山・水（仮の別色タイル）にぶつかったら移動をブロックする。
*   この時点で `npm run build` が通り、ブラウザ上で一連の基本動作が確認できるようにします。
